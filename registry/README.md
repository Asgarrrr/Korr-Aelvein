# Registry

Worldbuilding and narrative-design notes for Korr Aelvein.
**Not code, not technical docs** — the design of the world itself.

`docs/` (MAP.md, PROCGEN.md) speaks to the engine. `registry/` speaks to the world.

## Structure

| File | Contents |
|---|---|
| `world.md` | The universe: the abyss, the town, the mood, the moral climate, the chain |
| `systems.md` | Game-system design (progression, métiers, chain mechanics) |
| `vocabulary.md` | Words of the world — Korr, Aelvein, glyphs, songs (gameplay-bearing) |
| `story.md` | Events, characters, myths (stub) |
| `velm.md` | Single open notebook for ongoing design — sketches, fragments, threads. Content migrates out when it crystallizes |

## Writing convention

- A **firm decision**: written in plain prose in the right file.
- An **open question**: marked `> [TBD]` or `> [QUESTION]` in blockquote, in the context where it surfaces.
- A **rejected path**: `> [KILLED]` + the reason. Negative knowledge is load-bearing too — we record dead ends so we don't dig the same dry well twice.

This registry is a **working notebook**, not a published codex. It can contradict itself, evolve, be rewritten. When a decision lands in code (a tile name, an event format, a TypeBox schema), it migrates to `docs/` or `CLAUDE.md`.
